## Memory Consolidation Rules

**On correction:** Log the pattern immediately. Extract: the rule, the reason, how to apply it. Don't wait until session end.

**On task completion:** Ask: what was learned that changes future behavior? If nothing — fine. If something — write it. Convert mistakes into strict rules.

**On session start:** Review this file for patterns relevant to the current task. Don't repeat past mistakes.

**Bug autopsy:** After fixing a bug: explain WHY it happened and whether a category-level prevention exists. The fix is in the code; the lesson goes here.

**Dual-perspective review:** When evaluating own work, consider what a perfectionist would reject AND what a pragmatist would accept. Present both if the decision is non-obvious.

**Fresh eyes pass:** When testing own output: adopt a new-user persona. Walk through as if seeing the project for the first time. This is the right hemisphere checking the left hemisphere's work.
