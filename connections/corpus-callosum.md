# Corpus Callosum — Hemispheric Bridges

> The integration layer. Coordinates parallel and sequential activation between left and right hemispheres, and fans out to other regions when multi-domain convergence is needed.

---

## Integration Cycle (4 Phases)

Standard cycle for complex tasks. Phases can overlap. For simple tasks, compress 1 and 3 into quick checks.

1. **Explore (right)** — Divergent thinking. Scope the space. What patterns exist? What's adjacent? What would be surprising? What's the holistic view?
2. **Structure (left)** — Execute the analytical workflow. Verify each step. Apply evidence tiers. Run the domain skill sequentially.
3. **Review (right)** — Challenge the left hemisphere's output. What's missing? What doesn't fit? What would a contrarian say? Does the whole cohere? For major outputs: invoke your domain's challenge agent here. Accept valid corrections in place. The user sees one coherent output — the adversary is the review phase, not an appendix.
4. **Deliver (left)** — Verification checklist. Cleanup via falsificator. Clean final output.

**Execution note:** Phases 1 and 2 often run in parallel when the right hemisphere can scope while the left hemisphere executes the structured workflow. Phase 3 always follows Phase 2 (sequential dependency — left's output is right's input). Phase 4 follows Phase 3.

## Active Bridges

| Concept | Left (Analytical) | Right (Creative) |
|---------|-------------------|------------------|
| Architecture | Class diagrams, dependency maps | System interaction maps, metaphors |
| Debugging | Stack traces, error isolation | Error pattern visualization, systemic causes |
| Research analysis | Evidence tiers, mechanism chains | Cross-domain connections, what's missing |
| Planning | Gantt charts, checklists, feasibility | Concept flow, possibilities, risks |

## Co-activation Clusters

These region pairs fire together by default:

| Pair | Why |
|------|-----|
| cortex/left/ + cortex/right/ | All complex tasks — dual-hemisphere is the default |
| amygdala/ + falsificator | Urgent cleanup after crisis resolution |
| mirror-neurons + any region | Pattern matching runs continuously |
| default-mode/dmn + corpus-callosum | DMN candidate insights route here for left-hemisphere verification |
| hippocampus + any correction | Always log immediately when corrected |

## High-Frequency Circuits

```
Pattern A — Standard Processing (40% of tasks)
  thalamus → executive → left + right (parallel) → hippocampus/context-state

Pattern B — Crisis Mode (10% of tasks)
  thalamus → amygdala → executive → left → hippocampus/gotchas

Pattern C — Creative Exploration (25% of tasks)
  executive → right → corpus-callosum → left → hippocampus/context-state

Pattern F — DMN Insight (idle/transition/stuck)
  dmn → corpus-callosum → left (verify) → hippocampus/insights

Pattern D — Deep Analysis (20% of tasks)
  executive → left → corpus-callosum → right → hippocampus/gotchas

Pattern E — Reflection / Session End (5% of tasks)
  context-state → gotchas → executive → mirror-neurons
```

---

## Anti-Patterns (Do Not Route This Way)

- `left → left` without a creative check in between — marks a loop
- `amygdala → associations` — urgent issues need action, not exploration
- Any region → falsificator without executive awareness — cleanup needs context

---

## Routing After Integration

When corpus callosum completes a bridge:

| If integration result is… | Activate next |
|--------------------------|--------------|
| Implementation ready | cortex/left/ (sequential) |
| Still needs exploration | cortex/right/ (sequential) |
| Both modes satisfied | executive (return) |
| Novel cross-domain pattern | mirror-neurons (fan-out) + hippocampus/insights (fan-out) |
| Conflict between hemispheres | executive (sequential — needs decision) |
